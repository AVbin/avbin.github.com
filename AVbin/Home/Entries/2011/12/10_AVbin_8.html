Release: AVbin 8

 

AVbin 8

 

I am proud to announce AVbin 8, the first full release since 2008.


To all those who pointed me in the right direction on mailing lists and forums:  Thank you!!!  I would like to specifically mention Alex Holkner, who created this project in the first place, and Micah Richert, whose shoulders I stood on for this release.


AVbin 8 Highlights

  1. -Established which platforms were going to be supported going forward (added Win64 and OS X 64-bit Intel; dropped OS X 32-bit PPC)

  2. -Created development/build environments for each platform

  3. -Successfully compiled/released a build for each platform

  4. -Migrated from svn on Google Project to git on Github

  5. -Created a new project web site

  6. -28 commits

  7. -Approximately 15 issues closed...though mostly since they just don’t apply any more.


Known Issues with AVbin 8:

  1. -The version of FFmpeg used is one year old, and has known bugs and security problems

  2. -The AVbin binaries have not been tested internally.

  3. -The AVbin binaries have not been tested with Pyglet.

  4. -It is known that the build doesn’t actually work for a lot of file formats you would expect it to on at least one platform (win32)


The previous release was based on a 3-year old FFmpeg revision.  That revision had even more bugs and security problems than the newer one we used.  When I took over as the new maintainer almost exactly a month ago, I accepted the generous offer of Micah Richert to use his personal build he had put together a year ago as a starting point.  Since using the newest code was not a goal for this release, I simply used the exact revision that Micah used, to minimize the already considerable effort it was taking to get this project resurrected.


I’ve put together a Roadmap for things to accomplish for AVbin 9.  Check it out!


~ Nathan
